## ADDED Requirements

### Requirement: 新增問題單表單
系統 SHALL 提供問題單新增表單，包含以下 20 個欄位，並依提出/處理/覆測三個階段分組顯示：

**提出階段欄位**（新增時填寫）：
- 問題單編號（自動產生，唯讀）
- 提出日期（自動帶入今日，可修改）
- 提出人員（自動帶入登入者姓名，唯讀）
- 測試案例編號（下拉選單，來源：測試案例編號分頁）
- 測試功能項目（連動測試案例編號自動填入，可修改）
- 問題描述（多行文字輸入）
- 問題嚴重度（下拉：嚴重 / 功能無法運作 / 中 / 低 / 建議）
- 操作畫面/報表輔助說明附件（圖片上傳，可多張）

**處理階段欄位**（處理人員填寫）：
- 處理人員（下拉選單，來源：使用者白名單）
- 問題原因及處理方式（多行文字）
- 問題類型（下拉：Bug / 操作錯誤 / 需求 / 資料問題）
- 處理完成日期（日期選擇）

**覆測階段欄位**（測試/業務人員填寫）：
- 測試人員（下拉選單，來源：使用者白名單）
- 測試日期（日期選擇）
- 測試結果（下拉：OK / NG）
- 業務單位覆測人員（下拉選單）
- 覆測日期（日期選擇）
- 覆測結果（下拉：OK / NG）
- 問題狀態（自動計算或手動設定：Open / Closed / Reopen / 已刪除）
- 備註（多行文字）

#### Scenario: 提出人員新增問題單
- **WHEN** 登入使用者點擊「新增問題單」並填寫必填欄位後送出
- **THEN** 系統自動產生流水號編號、記錄提出日期與提出人員，問題單儲存至 Google Sheets，狀態設為「Open」

#### Scenario: 自動填入提出人員
- **WHEN** 使用者開啟新增問題單表單
- **THEN** 提出人員欄位自動帶入目前登入者姓名，且無法修改

#### Scenario: 測試案例連動功能項目
- **WHEN** 使用者選擇測試案例編號
- **THEN** 測試功能項目欄位自動帶入對應功能名稱

#### Scenario: 必填欄位驗證
- **WHEN** 使用者未填寫「問題描述」或「問題嚴重度」即送出
- **THEN** 系統顯示驗證錯誤提示，阻止送出

### Requirement: 編輯問題單表單
系統 SHALL 允許授權使用者編輯問題單各階段欄位。

#### Scenario: 提出人員編輯自己的問題單
- **WHEN** 提出人員開啟自己提出的問題單並修改內容
- **THEN** 修改儲存成功，Google Sheets 對應列更新

#### Scenario: 非提出人員無法修改提出階段欄位
- **WHEN** 非提出人員嘗試修改問題描述或嚴重度
- **THEN** 系統顯示「無權限修改」提示，欄位為唯讀狀態（管理員除外）
