## ADDED Requirements

### Requirement: 問題統計儀表板
系統 SHALL 提供儀表板頁面，以 Chart.js 圖表呈現以下統計資訊（不含已刪除問題單）：
1. 問題狀態分布（圓餅圖）：Open / Closed / Reopen 各數量
2. 問題嚴重度分布（長條圖）：嚴重 / 功能無法運作 / 中 / 低 / 建議 各數量
3. 問題類型分布（長條圖）：Bug / 操作錯誤 / 需求 / 資料問題 各數量
4. 每日新增問題單趨勢（折線圖）：近 30 天每日新增數量

**儀表板為登入後預設首頁**（導覽列順序：儀表板 → 問題單列表 → 新增問題單）。

#### Scenario: 載入儀表板
- **WHEN** 登入使用者完成登入
- **THEN** 系統自動導向儀表板頁，從 Apps Script 取得統計資料，以圖表方式呈現四種統計維度

#### Scenario: 儀表板數據即時反映
- **WHEN** 使用者新增或更新問題單後返回儀表板
- **THEN** 儀表板重新載入時顯示最新統計數據

### Requirement: 統計數字摘要
系統 SHALL 在儀表板頂部顯示關鍵數字卡片：總問題單數、Open 中數量、本週新增數量、本週關閉數量。

#### Scenario: 顯示數字摘要卡片（桌面）
- **WHEN** 儀表板在桌面上載入完成
- **THEN** 頁面頂部以橫排顯示 4 個數字卡片，各自顯示對應統計值

#### Scenario: 顯示數字摘要卡片（手機）
- **WHEN** 儀表板在手機上載入完成
- **THEN** 頁面頂部以 2x2 格顯示 4 個數字卡片，各自顯示對應統計值

### Requirement: 響應式儀表板佈局
系統 SHALL 以響應式佈局呈現儀表板，手機與桌面均可正常查閱統計資訊。

#### Scenario: 手機顯示儀表板
- **WHEN** 使用者以手機進入儀表板頁
- **THEN** 數字摘要卡片以 2x2 格排列，圖表以單欄垂直方式顯示，Chart.js 圖表自動縮放至螢幕寬度

#### Scenario: 桌面顯示儀表板
- **WHEN** 使用者以桌面瀏覽器進入儀表板頁
- **THEN** 數字摘要卡片以 1x4 橫排顯示，圖表以 2x2 格排列
